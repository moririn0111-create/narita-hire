# NARITA PRIVATE CAR — クラウドファンディングLP

成田空港エリアの訪日外国人向けハイヤー（貸切送迎）を開業するための
**クラウドファンディング挑戦ページ**です。GO風ダークデザイン・5言語対応・
応援登録フォーム（Googleフォーム連携）。集金は外部クラファン（Makuake等）で行う構成。

---

## フォルダ構成

```
~/narita-hire/
├── index.html              … サイト本体（1ページ完結）
├── css/style.css           … デザイン（色・サイズは :root を編集）
├── js/translations.js      … 全言語のテキスト（文言修正はここ）
├── js/i18n.js              … 言語切替の仕組み
├── js/booking.js           … 応援登録フォーム＋クラファンURL設定
├── README.md               … この説明書
└── .gitignore              … Git管理から外すファイル
```

## 1. まず動かして確認する

`index.html` をダブルクリックでブラウザ表示（サーバ不要）。
右上メニューで 英 / 簡体 / 繁体 / 韓 / 日 を切替。
Googleフォーム未設定の間は「デモモード」（送信内容はコンソール表示のみ・実送信なし）。

---

## 2. クラファン公開後にやること（リンクを差し込む）

Makuake / CAMPFIRE 等の公開ページが決まったら、`js/booking.js` の先頭：

```js
var CF_URL = "https://www.makuake.com/project/xxxxxx/";
```

を設定すると、ページ内の「応援する／このリターンで支援」ボタンが
すべてその外部クラファンページへ自動で飛ぶようになります（空なら内部スクロール）。

---

## 3. 応援登録（Googleフォーム連携）

メール先行登録を Googleスプレッドシートに自動でためる設定。**プログラム不要**。

### ステップ1：Googleフォームを作る
[forms.google.com](https://forms.google.com) で新規作成し、**5項目**を
「記述式」でこの順に追加：

1. Name
2. Email
3. Language
4. Interest
5. Notes（段落式でも可）

「回答」タブ → スプレッドシートアイコンで回答先シートを作成。
設定で「新しい回答をメール通知」をオンにすると即気づけます。

### ステップ2：送信先URLと項目IDを調べる
1. 編集画面 右上「︙」→「**事前入力したURLを取得**」
2. 各項目に適当な値を入れ「リンクを取得」
3. URLの `entry.123456789=...` の **数字** を5つメモ
4. 通常URLの `/viewform` を `/formResponse` に変えたものが送信先

### ステップ3：booking.js に貼り付ける
`js/booking.js` の `GOOGLE_FORM` を置き換え：

```js
var GOOGLE_FORM = {
  action: "https://docs.google.com/forms/d/e/【フォームID】/formResponse",
  entries: {
    name:     "entry.111111",
    email:    "entry.222222",
    language: "entry.333333",
    interest: "entry.444444",
    notes:    "entry.555555"
  }
};
```

`REPLACE` が1つでも残るとデモモードのまま（誤送信防止）。

## 4. Vercel公開（GitHub経由）

このフォルダはGitリポジトリ化済みです。GitHubに上げてVercelと連携すると、
**以後は更新を保存するだけで自動で再公開**されます。手順は会話の指示どおり。

更新の反映方法（公開後）：

```
cd ~/narita-hire
git add -A
git commit -m "文言修正など"
git push
```
→ Vercelが自動で再ビルド・再公開。

## 5. 重要な法律メモ（A案：自前で許可取得）

- 本ページは「**応援登録の受付のみ**」。決済・予約確定はしません。
- 返礼の**ライドチケットは「サービス開始（運送事業許可の取得）後に有効」**と明記済み。
- クラファンは **All-or-Nothing（目標未達なら全額返金）** を推奨。
  許認可が必要な事業はプラットフォーム審査があるため、申請状況を正直に説明すること。
- 許可前に有償運送・本決済を行わないこと（白タク＝道路運送法違反）。
